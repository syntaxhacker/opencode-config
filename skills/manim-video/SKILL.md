# Manim Video Production Pipeline

## Creative Standard

This is educational cinema. Every frame teaches. Every animation reveals structure.

**Before writing a single line of code**, articulate the narrative arc. What misconception does this correct? What is the "aha moment"? What visual story takes the viewer from confusion to understanding? The user's prompt is a starting point — interpret it with pedagogical ambition.

**Geometry before algebra.** Show the shape first, the equation second. Visual memory encodes faster than symbolic memory. When the viewer sees the geometric pattern before the formula, the equation feels earned.

**First-render excellence is non-negotiable.** The output must be visually clear and aesthetically cohesive without revision rounds. If something looks cluttered, poorly timed, or like "AI-generated slides," it is wrong.

**Opacity layering directs attention.** Never show everything at full brightness. Primary elements at 1.0, contextual elements at 0.4, structural elements (axes, grids) at 0.15. The brain processes visual salience in layers.

**Breathing room.** Every animation needs `self.wait()` after it. The viewer needs time to absorb what just appeared. Never rush from one animation to the next. A 2-second pause after a key reveal is never wasted.

**Cohesive visual language.** All scenes share a color palette, consistent typography sizing, matching animation speeds. A technically correct video where every scene uses random different colors is an aesthetic failure.

## Prerequisites

Requires: Python 3.10+, Manim Community Edition v0.20+ (`pip install manim`), LaTeX (`texlive-full` on Linux, `mactex` on macOS), and ffmpeg. Reference docs tested against Manim CE v0.20.1.

## Stack

Multi-file project with sub-agent parallelism. No browser, no Node.js, no GPU required.

| Layer | Tool | Purpose |
|-------|------|---------|
| Orchestration | Task sub-agents | Parallel scene creation — one sub-agent per scene |
| Core | Manim Community Edition | Scene rendering, animation engine |
| Math | LaTeX (texlive/MiKTeX) | Equation rendering via `MathTex` |
| Video I/O | ffmpeg | Scene stitching, format conversion, audio muxing |
| TTS | ElevenLabs / Qwen3-TTS (optional) | Narration voiceover |

## Pipeline

```
PLAN --> CONFIG --> DECOMPOSE & CODE (parallel) --> RENDER --> STITCH --> REVIEW
```

1. **PLAN** — Write `plan.md` with narrative arc, scene list, visual elements, color palette, voiceover script
2. **CONFIG** — Create `config.py` with shared constants (colors, fonts, helpers)
3. **DECOMPOSE & CODE** — Dispatch one `task` sub-agent per scene in parallel. Each writes an independent file in `scenes/`
4. **RENDER** — Render each scene file individually: `manim -ql scenes/scene1_xxx.py Scene1Xxx`
5. **STITCH** — ffmpeg concat of scene clips into `final.mp4`
6. **REVIEW** — Render preview stills, verify against plan, adjust

## Project Structure

```
project-name/

```

### Animation Speed

| Context | run_time | self.wait() after |
|---------|----------|-------------------|
| Title/intro appear | 1.5s | 1.0s |
| Key equation reveal | 2.0s | 2.0s |
| Transform/morph | 1.5s | 1.5s |
| Supporting label | 0.8s | 0.5s |
| FadeOut cleanup | 0.5s | 0.3s |
| "Aha moment" reveal | 2.5s | 3.0s |

### Typography Scale

| Role | Font size | Usage |
|------|-----------|-------|
| Title | 48 | Scene titles, opening text |
| Heading | 36 | Section headers within a scene |
| Body | 30 | Explanatory text |
| Label | 24 | Annotations, axis labels |
| Caption | 20 | Subtitles, fine print |

### Fonts

**Use monospace fonts for all text.** Manim's Pango renderer produces broken kerning with proportional fonts at all sizes. See `references/visual-design.md` for full recommendations.

```python
MONO = "Menlo"  # define once at top of file

Text("Fourier Series", font_size=48, font=MONO, weight=BOLD)  # titles
Text("n=1: sin(x)", font_size=20, font=MONO)                  # labels
MathTex(r"\nabla L")                                            # math (uses LaTeX)
```

Minimum `font_size=18` for readability.

### Per-Scene Variation

Never use identical config for all scenes. For each scene:
- **Different dominant color** from the palette
- **Different layout** — don't always center everything
- **Different animation entry** — vary between Write, FadeIn, GrowFromCenter, Create
- **Different visual weight** — some scenes dense, others sparse

## Workflow

### Step 1: Plan (plan.md)

Before any code, write `plan.md` with a numbered scene list. Each scene gets a name, purpose, layout, and estimated duration. See `references/scene-planning.md` for the comprehensive template.

From `plan.md`, extract the scene list into a concrete dispatch table:

```
Scene 1: scene1_introduction.py        — Hook, title card, pose the question
Scene 2: scene2_visual_intuition.py    — Geometric intuition build-up
Scene 3: scene3_equation_derivation.py — Step-by-step equation reveal
Scene 4: scene4_application.py         — Real-world example or result
Scene 5: scene5_conclusion.py          — Summary and implications
```

### Step 2: Create config.py

Write `config.py` once. Every scene file imports from it — this guarantees cross-scene visual consistency.

```python
from manim import *

BG = "#1C1C1C"
PRIMARY = "#58C4DD"
SECONDARY = "#83C167"
ACCENT = "#FFFF00"
HIGHLIGHT = "#FF6B6B"
DIM = "#444444"
MONO = "Menlo"

TITLE_SIZE = 48
HEADING_SIZE = 36
BODY_SIZE = 30
LABEL_SIZE = 24
CAPTION_SIZE = 20

FAST = 0.8
NORMAL = 1.5
SLOW = 2.5

def clean_exit(scene):
    scene.play(FadeOut(Group(*scene.mobjects), run_time=0.5))
    scene.wait(0.3)
```

### Step 3: Decompose & Code (parallel sub-agents)

Dispatch **one `task` sub-agent per scene** in a single message. All sub-agents run in parallel.

Each sub-agent receives:
- The scene's assignment from `plan.md` (purpose, layout, visual elements, animation sequence)
- The full `config.py` contents (to import constants from)
- Which reference docs to read (tailored to the scene's needs)

#### Sub-agent dispatch template

Use this exact prompt structure for each `task` call. Replace the `{braces}` with scene-specific values:

```
Write a Manim Community Edition scene file at {project_dir}/scenes/{filename}.

READ THESE FILES FIRST:
- {project_dir}/config.py          — shared constants to import
- {skill_dir}/references/{ref1}.md  — API reference for this scene type
- {skill_dir}/references/{ref2}.md  — additional reference if needed

SCENE ASSIGNMENT (from plan.md):
Scene {N}: {scene_name}
Purpose: {one-sentence description}
Layout: {FULL_CENTER / LEFT_RIGHT / TOP_BOTTOM / GRID / PROGRESSIVE / ANNOTATED_DIAGRAM}
Estimated duration: {Ns}

VISUAL ELEMENTS:
- {mobject description, position, color}
- ...

ANIMATION SEQUENCE:
1. {animation} — {what it reveals} (~Ns)
2. {animation} — {what it reveals} (~Ns)
- ...

SUBTITLE TEXT:
"{subtitle for this scene}"

REQUIREMENTS:
- Import constants from config: from config import BG, PRIMARY, SECONDARY, ACCENT, HIGHLIGHT, DIM, MONO, clean_exit
- Import Manim objects directly: from manim import *
- Set self.camera.background_color = BG at the start of construct()
- Use MONO font for ALL Text() objects
- Minimum font_size=18
- Add self.add_subcaption() or subcaption= on every self.play() call
- self.wait() after every reveal (1.5s for key moments, 0.5s for supporting)
- buff >= 0.5 on all .to_edge() calls
- Use FadeOut(Group(*self.mobjects)) at scene end, NOT FadeOut individual objects
- Use Group (not VGroup) when mixing Text with shapes
- Use raw strings (r"...") for all LaTeX
- For equations, use FadeTransform over Transform for dissimilar shapes
- Opacity layering: primary=1.0, context=0.4, structural=0.15
- Vary animation types: don't use Write() for everything
- Keep max 5-6 elements visible at once

Write ONLY the scene file. Do not create config.py or script.py.
```

#### Choosing reference docs per scene

| Scene type | References to include |
|------------|----------------------|
| Title / intro / text-heavy | `animations.md`, `mobjects.md`, `visual-design.md` |
| Equation derivation | `equations.md`, `animations.md`, `decorations.md` |
| Algorithm / data viz | `graphs-and-data.md`, `animations.md`, `updaters-and-trackers.md` |
| Architecture / system diagram | `mobjects.md`, `decorations.md`, `animations.md` |
| 3D / spatial | `camera-and-3d.md`, `mobjects.md`, `animations.md` |
| Linear algebra / transforms | `camera-and-3d.md`, `animations.md`, `equations.md` |
| Paper explainer | `paper-explainer.md`, `equations.md`, `graphs-and-data.md` |

#### Dispatch example (5 scenes in parallel)

In a single message, issue 5 `task` calls:

```
task(description="Write scene 1: introduction", subagent_type="general", prompt="<template filled for scene1>")
task(description="Write scene 2: visual intuition", subagent_type="general", prompt="<template filled for scene2>")
task(description="Write scene 3: equation derivation", subagent_type="general", prompt="<template filled for scene3>")
task(description="Write scene 4: application", subagent_type="general", prompt="<template filled for scene4>")
task(description="Write scene 5: conclusion", subagent_type="general", prompt="<template filled for scene5>")
```

All 5 run concurrently. For a 10-scene video, dispatch 10 sub-agents.

### Step 4: Render (per-file)

**Important:** Manim CE uses `obj.__module__` checks to discover scenes, so imported Scene subclasses in `script.py` are NOT discovered. Each scene file must be rendered individually from its own file.

Each scene file must add the project root to `sys.path` so it can find `config.py`:

```python
import sys, os; sys.path.insert(0, os.path.join(os.path.dirname(__file__), ".."))
from manim import *
from config import *
```

Render each scene from its file:

```bash
manim -ql scenes/scene1_introduction.py Scene1_Introduction
manim -ql scenes/scene2_core_concept.py Scene2_CoreConcept
manim -ql scenes/scene3_equation_derivation.py Scene3_EquationDerivation
manim -ql scenes/scene4_application.py Scene4_Application
manim -ql scenes/scene5_conclusion.py Scene5_Conclusion
```

Or render all with a loop:

```bash
for f in scenes/scene*.py; do
  scene=$(grep -oP 'class \K\w+(?=\(Scene\))' "$f")
  manim -ql "$f" "$scene"
done
```

### Step 5: Stitch

Media output goes to `media/videos/<filename_without_ext>/480p15/<SceneClass>.mp4`. Build `concat.txt` with the actual paths:

```bash
cat > concat.txt << 'EOF'
file 'media/videos/scene1_introduction/480p15/Scene1_Introduction.mp4'
file 'media/videos/scene2_core_concept/480p15/Scene2_CoreConcept.mp4'
file 'media/videos/scene3_equation_derivation/480p15/Scene3_EquationDerivation.mp4'
file 'media/videos/scene4_application/480p15/Scene4_Application.mp4'
file 'media/videos/scene5_conclusion/480p15/Scene5_Conclusion.mp4'
EOF
ffmpeg -y -f concat -safe 0 -i concat.txt -c copy final.mp4
```

### Step 6: Review

```bash
manim -ql --format=png -s scenes/scene3_equation_derivation.py Scene3_EquationDerivation  # preview still
```

Watch the stitched `final.mp4` at 1x speed. Check transitions, pacing, text readability. Fix individual scenes by re-dispatching a single sub-agent for that scene, then re-render and re-stitch only the changed scene.

## Critical Implementation Notes

### Raw Strings for LaTeX
```python
# WRONG: MathTex("\frac{1}{2}")
# RIGHT:
MathTex(r"\frac{1}{2}")
```

### buff >= 0.5 for Edge Text
```python
label.to_edge(DOWN, buff=0.5)  # never < 0.5
```

### FadeOut Before Replacing Text
```python
self.play(ReplacementTransform(note1, note2))  # not Write(note2) on top
```

### Never Animate Non-Added Mobjects
```python
self.play(Create(circle))  # must add first
self.play(circle.animate.set_color(RED))  # then animate
```

## Performance Targets

| Quality | Resolution | FPS | Speed |
|---------|-----------|-----|-------|
| `-ql` (draft) | 854x480 | 15 | 5-15s/scene |
