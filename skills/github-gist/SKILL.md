# GitHub Gist Creator

Create, save locally, and publish GitHub Gists from any markdown content, code snippets, or text — all from the CLI using `gh`.

## Input

Ask the user for (or infer from context):

1. **Content** — markdown text, code snippet, or file to gist. Can be:
   - Inline content provided in the message
   - A local file path
   - Content generated by another skill (e.g. paper-to-gist, mermaid-diagram)
2. **Filename** (optional) — name for the gist file (with extension). Default: `<topic-slug>.md`
3. **Description** (optional) — gist description shown on GitHub. Default: infer from content title/first heading
4. **Visibility** (optional) — `public` or `secret`. Default: `public`
5. **Multiple files** (optional) — gist can contain multiple files. Provide as list of `{filename: path_or_content}` pairs.

## Workflow

### Step 1: Prepare Content

- If the user provides content directly, write it to a local `.md` file first (in the working directory)
- If the user provides a file path, verify it exists
- If content is generated from another skill, use that output directly

### Step 2: Create the Gist

**Single file:**
```bash
gh gist create <file.md> -p -d "Description here"
```

**Multiple files:**
```bash
gh gist create file1.md file2.py file3.json -p -d "Description here"
```

**From stdin (inline content):**
```bash
gh gist create -p -f "filename.md" -d "Description here" - <<'EOF'
<content here>
EOF
```

Flags:
- `-p, --public` — make gist public (omit for secret/unlisted)
- `-d, --desc "text"` — description
- `-f, --filename "name.ext"` — filename when reading from stdin
- `-w, --web` — open in browser after creation

### Step 3: Return Result

Return the gist URL to the user in the format:
```
https://gist.github.com/<username>/<gist-id>
```

## CLI Reference

```bash
# Create public gist from file
gh gist create myfile.md --public --desc "My guide"

# Create secret gist from stdin
echo "hello world" | gh gist create - --filename "hello.py"

# Create gist with multiple files
gh gist create a.py b.js c.ts -p -d "Multi-file gist"

# List your gists
gh gist list --limit 10

# View a gist
gh gist view <gist-id>

# Edit a gist
gh gist edit <gist-id> file.md

# Delete a gist
gh gist delete <gist-id>
```

## Integration with Other Skills

This skill is designed to work as a **final publishing step** for other skills:

| Source Skill | Typical Content | Filename |
|---|---|---|
| `paper-to-gist` | Paper deep dive markdown | `<topic>-deep-dive.md` |
| `mermaid-diagram` | Mermaid source file (`.mmd`) | `<topic>.mmd` |
| Any code task | Code snippets, configs, scripts | `<topic>.py` / `.ts` / `.json` |

When another skill generates content and the user says "gist it" or "upload to gist", use this skill to publish.

## Quality Checklist

Before publishing, verify:

- [ ] Content is saved locally before gisting (in case of failure)
- [ ] Filename has the correct extension (`.md`, `.py`, `.json`, etc.)
- [ ] Description is meaningful (not empty)
- [ ] No secrets/API keys in the content (grep for `key`, `token`, `secret`, `password`)
- [ ] The gist URL is returned to the user after creation

## Common Issues & Fixes

| Issue | Fix |
|---|---|
| `gh: not found` | Install GitHub CLI: `brew install gh` then `gh auth login` |
| `not logged in` | Run `gh auth login` to authenticate |
| Flag error (`--title` not valid) | Use `-d` / `--desc` for description, not `--title` |
| Binary file (PNG, etc.) | Gists support text only — upload the source file (e.g. `.mmd`) with compile instructions |
| Content too large | GitHub gist limit is ~1MB per file — split into multiple gists if needed |
| Accidental secret leak | Use `gh gist delete <id>` immediately, then rotate the leaked credential |
